# 06-测试与验收规范

## 1. 验收目标
1. URL 跳转能力可用。
2. 命中率 >= 95%。
3. 多命中选择与未命中提示符合 PRD。

来源：`docs/prd.md`《4.验收标准》

## 2. 测试层次

### 2.1 单元测试
1. 路径拼接测试（类级 + 方法级）。
2. URL 标准化测试（context-path、尾斜杠、重复斜杠、大小写）。
3. 排序规则测试（5级排序稳定性）。

### 2.2 集成测试
1. 命令触发 -> 输入 -> 匹配 -> 跳转完整链路。
2. 多命中选择链路。
3. 未命中提示链路。

### 2.3 样例仓库回归
1. 在样例仓库清单中逐项执行。
2. 输出命中率报告。

## 3. 命中率统计口径
1. 统计单位：一次有效输入 URL。
2. 命中成功：跳转到期望 `Controller#method`。
3. 命中失败：未命中或跳转到错误方法。
4. 通过标准：`命中成功数 / 总样本数 >= 95%`。

来源：`docs/prd.md`《4.验收标准》

## 4. 失败分类定义
1. 解析失败：未正确构建 endpoint 索引。
2. 匹配失败：有索引但排序/匹配未选中目标。
3. 跳转失败：已命中但打开文件或定位范围错误。

## 5. 验收输出物
1. 测试报告（含样本、结果、失败分类）。
2. 命中率统计结果。
3. 回归问题修复记录。
