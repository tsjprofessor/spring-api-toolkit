# 01-代码组织结构

## 1. 目标
基于 PRD 的极简 MVP 范围，建立“可快速落地 URL 跳转”的最小代码组织结构，避免过度设计。

来源：PRD《1.产品目标》《3.1 本期必做（In Scope）》

## 2. 推荐目录结构（MVP）
```text
src/
  extension.ts                  # 激活入口、命令注册
  commands/
    goto-by-url.command.ts      # URL 输入与跳转流程编排
  indexer/
    endpoint-indexer.ts         # 扫描源码并构建 endpoint 索引
    endpoint-model.ts           # Endpoint 领域模型
  parser/
    spring/
      class-mapping-parser.ts   # 类级注解解析
      method-mapping-parser.ts  # 方法级注解解析
  matcher/
    url-matcher.ts              # URL 与 endpoint 的匹配逻辑
  navigation/
    method-navigator.ts         # 打开文件并定位到方法
  ui/
    quick-pick.ts               # 多命中选择
  infra/
    logger.ts                   # 日志封装
    errors.ts                   # 领域错误定义
```

来源：PRD《3.1 F1 URL 到方法跳转》《4.交互说明》《6.非功能要求》

## 3. 分层约束
1. `commands` 只负责流程编排，不做解析实现。
2. `parser` 只做注解/路径解析，不做 UI。
3. `matcher` 只做匹配判定，不直接操作编辑器。
4. `navigation` 只做跳转行为，不感知解析细节。

来源：PRD《6.非功能要求》（不阻塞主线程、可维护）

## 4. 不在本期实现的目录
1. `http-client/`
2. `services-tree/`
3. `json-tools/`

来源：PRD《3.2 本期不做（Out of Scope）》
