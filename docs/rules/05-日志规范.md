# 05-日志规范

## 1. 目标
支持调试“为什么没命中”和“为什么跳转失败”。

来源：PRD《5.验收标准》《8.风险与应对》

## 2. 日志级别
1. `info`：命令触发、索引完成、命中成功。
2. `warn`：未命中、多命中、单文件解析失败。
3. `error`：跳转失败、不可恢复异常。

## 3. 关键日志字段
1. `inputUrl`
2. `httpMethod`（如用户输入）
3. `candidateCount`
4. `selectedEndpoint`
5. `costMs`

## 4. 输出规范
1. 统一使用插件输出通道，例如 `Qoder REST Toolkit`。
2. 每条日志附时间戳与流程阶段（scan/match/navigate）。

来源：PRD《6.非功能要求》（性能与稳定性）
