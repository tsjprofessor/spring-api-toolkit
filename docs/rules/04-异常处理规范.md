# 04-异常处理规范

## 1. 目标
保证 URL 跳转流程在异常情况下可恢复、可提示、可定位。

来源：PRD《3.1 F1》《4.交互说明》

## 2. 异常分层
1. `ParseError`：注解或路径解析失败。
2. `MatchError`：匹配流程异常。
3. `NavigateError`：文件打开或定位失败。

## 3. 用户提示规范
1. 未命中：提示“未匹配到接口方法”。
2. 多命中：弹出选择列表，不视为错误。
3. 异常：提示简短错误 + 引导查看输出日志。

来源：PRD《3.1 F1 命中逻辑》

## 4. 处理原则
1. 不吞异常，必须记录日志。
2. 不在 UI 层暴露堆栈细节。
3. 不因单文件解析失败而中断全局索引。

来源：PRD《6.非功能要求》
