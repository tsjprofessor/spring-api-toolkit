# 08-日志与诊断手册

## 1. 目标
为开发和联调提供统一诊断路径，快速定位“未命中、多命中、跳转错误”的根因。

来源：`docs/rules/05-日志规范.md`、`docs/prd.md`《4.验收标准》

## 2. 输出通道与关键字段

### 2.1 输出通道
统一输出到插件通道：`Qoder REST Toolkit`

### 2.2 关键字段
1. `stage`：`scan` / `parse` / `match` / `navigate`
2. `inputUrl`
3. `normalizedUrl`
4. `httpMethod`
5. `candidateCount`
6. `selectedEndpoint`
7. `costMs`
8. `errorCode`
9. `errorMessage`

来源：`docs/rules/05-日志规范.md`

## 3. 常见问题诊断流程

### 3.1 问题：输入 URL 无命中
排查步骤：
1. 看 `match` 阶段日志是否拿到 `candidateCount=0`。
2. 检查 `scan` 阶段是否扫描到预期模块路径（`src/main/java`）。
3. 检查 `parse` 阶段是否正确识别类级/方法级注解。
4. 检查 `normalizedUrl` 是否被 `contextPath` 或斜杠归一规则改写。

可能原因：
1. 扫描目录配置错误。
2. endpoint 未被解析。
3. URL 标准化规则与输入不一致。

### 3.2 问题：命中了多个但目标不在首位
排查步骤：
1. 查看 `match` 阶段输出的候选排序分值。
2. 核对是否按已确认排序规则执行。
3. 检查输入是否缺失 METHOD，导致候选扩大。

可能原因：
1. 排序实现偏离规则。
2. 规范化路径相同导致候选并列过多。

来源：`docs/domain-knowledge/03-待确认问题清单.md`《1.7 多命中排序规则》

### 3.3 问题：跳转到了错误位置
排查步骤：
1. 查看 `selectedEndpoint` 是否正确。
2. 查看 `navigate` 阶段 `filePath` 与 `range` 是否正确。
3. 确认源码变更后索引是否刷新。

可能原因：
1. 索引过期。
2. 方法 range 计算错误。
3. 文件内容变化导致位置信息失效。

### 3.4 问题：快捷键无响应
排查步骤：
1. 在 IDE Keymap 中检查 `Ctrl+\\` 是否被其他命令覆盖。
2. 尝试备用快捷键 `Ctrl+Alt+N`。
3. 直接从命令面板执行 `REST Toolkit: Go To Endpoint By URL`。

可能原因：
1. 快捷键冲突。
2. 快捷键被用户改绑。

来源：`docs/rules/07-快捷键冲突策略.md`

## 4. 诊断日志模板

```text
[info] stage=match inputUrl=/user/add normalizedUrl=/user/add httpMethod=POST candidateCount=3 costMs=12
[info] stage=match selectedEndpoint=UserController#add fullPath=/user/add
[info] stage=navigate filePath=.../UserController.java range=120:5 costMs=3
```

## 5. 升级处理策略
1. 首次排查优先看 `scan -> parse -> match -> navigate` 链路是否断点。
2. 若连续出现同类失败，沉淀为回归用例并加入验收样例。
3. 若影响主链路稳定性，先回退到上一个稳定版本策略。
