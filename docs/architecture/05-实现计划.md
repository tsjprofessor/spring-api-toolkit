# 05-实现计划

## 1. 目标
在 `0.1.0` 版本内完成“输入 URL -> 定位方法定义”的最小闭环能力。

来源：`docs/prd.md`《1.产品目标》《3.1 本期必做》

## 2. 任务拆解

### 任务1：建立 Endpoint 领域模型
1. 定义 `Endpoint`：`module`、`filePath`、`className`、`methodName`、`httpMethod`、`classPath`、`methodPath`、`fullPath`、`range`。
2. 输出：统一的数据结构供索引、匹配、跳转复用。
3. 完成标准：索引结果可序列化并可被 matcher 直接消费。

### 任务2：实现源码扫描器（多模块）
1. 扫描范围：各模块 `src/main/java`。
2. 排除：`src/test`、`target`、`build`。
3. 输出：候选 Controller 文件列表。
4. 完成标准：多模块项目能产出稳定扫描结果。

来源：`docs/prd.md`《3.1 F4 扫描范围》

### 任务3：实现 Spring 路由解析器
1. 解析类级注解与方法级注解。
2. 合成 `fullPath`（类级 + 方法级）。
3. 输出：`Endpoint[]`。
4. 完成标准：`/user` + `/add` 可正确合成为 `/user/add`。

来源：`docs/prd.md`《3.1 F1/F2》

### 任务4：实现 URL 标准化器
1. 支持 `context-path` 前缀。
2. 处理尾斜杠。
3. 处理重复斜杠。
4. 处理大小写。
5. 完成标准：同一路径不同写法归一后命中一致。

来源：`docs/prd.md`《3.1 F3 URL 标准化处理》

### 任务5：实现匹配与排序
1. 生成候选集合。
2. 执行已确认排序规则：
- 完整路径精确匹配优先
- 规范化后精确匹配优先
- 前缀匹配优先于包含匹配
- 路径短优先
- `Controller#method` 字典序稳定排序
3. 完成标准：多命中可稳定复现排序结果。

来源：`docs/domain-knowledge/03-待确认问题清单.md`《1.7 多命中排序规则》

### 任务6：实现命令与输入交互
1. 命令：`restToolkit.gotoByUrl`。
2. 快捷键：`Ctrl+\\`/`Cmd+\\` + 备用 `Ctrl+Alt+N`/`Cmd+Alt+N`。
3. 完成标准：快捷键可唤醒输入框并触发匹配流程。

来源：`docs/prd.md`《3.1 F5 交互与展示》

### 任务7：实现跳转与多命中选择
1. 唯一命中：直接跳转。
2. 多命中：列表选择后跳转。
3. 未命中：提示“未匹配到接口方法”。
4. 完成标准：三类分支都可用且行为一致。

来源：`docs/prd.md`《3.1 F1 命中逻辑》

### 任务8：实现日志与错误处理
1. 记录输入、候选数量、耗时、最终选择。
2. 异常可追踪，用户提示简洁。
3. 完成标准：失败场景可定位问题。

来源：`docs/rules/04-异常处理规范.md`、`docs/rules/05-日志规范.md`

### 任务9：回归测试与命中率验收
1. 执行样例仓库批量验证。
2. 产出命中率统计报告。
3. 完成标准：命中率 >= 95%。

来源：`docs/prd.md`《4.验收标准》

## 3. 交付物
1. 可安装调试的插件版本（0.1.0）。
2. 命中率统计报告。
3. 问题清单与修复记录。
