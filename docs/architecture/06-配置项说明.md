# 06-配置项说明

## 1. 文档目标
定义 `0.1.0` 版本配置项，明确默认值、作用范围与生效方式，避免实现阶段产生歧义。

来源：`docs/prd.md`《3.1 F3 URL 标准化处理》《3.1 F4 扫描范围》《3.1 F5 交互与展示》

## 2. 配置项总览

| 配置键 | 类型 | 默认值 | 作用范围 | 说明 |
|---|---|---|---|---|
| `restToolkit.scanRoots` | `string[]` | `['src/main/java']` | 工作区级 | 参与索引的源码根目录（多模块按模块根拼接） |
| `restToolkit.excludeGlobs` | `string[]` | `['**/src/test/**','**/target/**','**/build/**']` | 工作区级 | 扫描排除规则 |
| `restToolkit.contextPath` | `string` | `''` | 工作区级 | URL 匹配时的 context-path 前缀，例如 `/api` |
| `restToolkit.normalizeTrailingSlash` | `boolean` | `true` | 工作区级 | 是否归一化尾斜杠 |
| `restToolkit.normalizeRepeatedSlash` | `boolean` | `true` | 工作区级 | 是否归一化重复斜杠 |
| `restToolkit.caseInsensitiveMatch` | `boolean` | `true` | 工作区级 | 是否大小写不敏感匹配 |
| `restToolkit.primaryHotkeyEnabled` | `boolean` | `true` | 用户级 | 是否启用主快捷键 `Ctrl+\\` / `Cmd+\\` |
| `restToolkit.fallbackHotkeyEnabled` | `boolean` | `true` | 用户级 | 是否启用备用快捷键 `Ctrl+Alt+N` / `Cmd+Alt+N` |
| `restToolkit.maxCandidates` | `number` | `100` | 工作区级 | 多命中候选最大展示数量 |
| `restToolkit.logLevel` | `'info' \| 'warn' \| 'error'` | `info` | 用户级 | 插件日志级别 |

## 3. 配置项详细说明

### 3.1 `restToolkit.scanRoots`
1. 仅用于索引扫描，不影响跳转行为。
2. MVP 默认只扫描 `src/main/java`。
3. 多模块项目由插件自动识别模块根并拼接该路径。

来源：`docs/prd.md`《3.1 F4 扫描范围》

### 3.2 `restToolkit.excludeGlobs`
1. 默认排除测试代码与构建输出目录。
2. 用户可增加自定义排除路径（如 `generated`）。

来源：`docs/prd.md`《3.1 F4 扫描范围》

### 3.3 `restToolkit.contextPath`
1. 输入 URL 与 endpoint 对比前，先做 `contextPath` 归一处理。
2. 例如 `contextPath=/api` 时，输入 `/api/user/add` 可匹配 `/user/add` endpoint。

来源：`docs/prd.md`《3.1 F3 URL 标准化处理》

### 3.4 URL 标准化开关
1. `normalizeTrailingSlash=true`：`/user/add/` -> `/user/add`
2. `normalizeRepeatedSlash=true`：`/user//add` -> `/user/add`
3. `caseInsensitiveMatch=true`：`/User/Add` 与 `/user/add` 视为可匹配

来源：`docs/prd.md`《3.1 F3 URL 标准化处理》

### 3.5 快捷键相关开关
1. `primaryHotkeyEnabled` 控制主快捷键是否启用。
2. `fallbackHotkeyEnabled` 控制备用快捷键是否启用。
3. 实际冲突优先按 IDE Keymap 用户配置生效。

来源：`docs/prd.md`《3.1 F5 交互与展示》、`docs/rules/07-快捷键冲突策略.md`

### 3.6 `restToolkit.maxCandidates`
1. 控制多命中列表的显示上限，避免候选过多造成交互卡顿。
2. 超出上限时仅显示前 N 条（按已确认排序规则截断）。

来源：`docs/domain-knowledge/03-待确认问题清单.md`《1.7 多命中排序规则》

### 3.7 `restToolkit.logLevel`
1. `info`：记录完整链路日志。
2. `warn`：仅记录告警与错误。
3. `error`：仅记录错误。

来源：`docs/rules/05-日志规范.md`

## 4. 生效机制
1. 工作区配置优先于用户全局配置。
2. 配置更新后，扫描相关项需要触发索引刷新才完全生效。
3. 快捷键配置变更按 IDE 行为即时生效。

## 5. 验收检查清单
1. 修改 `contextPath` 后，匹配结果符合预期。
2. 关闭大小写不敏感后，大小写错误输入不可命中。
3. 修改 `scanRoots` 与 `excludeGlobs` 后，索引目标变化正确。
4. 主/备快捷键开关行为符合预期。
