# 07-快捷键冲突策略

## 1. 目标
在保留高效唤起体验的同时，避免与 IDE 默认快捷键冲突影响日常开发。

## 2. 默认快捷键策略
1. 主快捷键：`Ctrl+\\`（Mac: `Cmd+\\`）。
2. 备用快捷键：`Ctrl+Alt+N`（Mac: `Cmd+Alt+N`）。

来源：`docs/prd.md`《3.1 F5 交互与展示》

## 3. 冲突处理策略
1. 用户可在 IDE Keymap 中自定义绑定。
2. 保持备用快捷键长期可用，避免因主快捷键冲突导致功能不可达。
3. 命令面板始终可触发 `REST Toolkit: Go To Endpoint By URL`。

## 4. 文档与提示规范
1. README 中明确两套默认快捷键。
2. 遇到用户反馈冲突时，优先指导改绑到备用键。

## 5. 验收要求
1. 两套快捷键均可唤起输入框。
2. 命令面板触发行为与快捷键一致。
