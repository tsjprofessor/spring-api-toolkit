# 02-模块职责说明

## 1. Command 模块
职责：接收用户命令，驱动主流程，不承载解析细节。

来源：PRD《3.1 F1》《4.交互说明》

## 2. Indexer 模块
职责：扫描 Java（及可选 Kotlin）源码，构建 endpoint 索引。

来源：PRD《3.1 F2》《6.非功能要求》

## 3. Parser 模块
职责：解析 Spring 注解并生成类级/方法级路径信息。

来源：PRD《3.1 F2 最小框架支持》

## 4. Matcher 模块
职责：接收用户输入 URL，输出命中结果集合。

来源：PRD《3.1 F1 命中逻辑》

## 5. Navigator 模块
职责：根据命中结果打开文件并定位方法定义。

来源：PRD《4.交互说明》《5.验收标准》

## 6. UI 模块（Quick Pick）
职责：处理多命中用户选择。

来源：PRD《3.1 F1 多个命中时选择》
