# 03-模块划分

## 1. 划分原则
1. 以“URL 跳转主链路”划分，非主链路功能不进入 MVP。
2. 解析、匹配、跳转分离，便于后续扩展 JAX-RS/Kotlin。

来源：PRD《1.产品目标》《3.2 本期不做》《7.版本计划》

## 2. 模块清单
1. 输入模块：URL 输入获取。
2. 索引模块：源码扫描与缓存。
3. 解析模块：Spring 注解解析。
4. 匹配模块：METHOD+PATH 判定。
5. 跳转模块：文件定位。
6. 选择模块：多命中交互。

来源：PRD《3.1 F1/F2》《4.交互说明》

## 3. 扩展预留
1. 解析模块预留 JAX-RS 解析器插槽。
2. 索引模块预留 Kotlin 完整支持。

来源：PRD《7.版本计划》
