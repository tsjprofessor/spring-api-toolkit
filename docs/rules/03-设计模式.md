# 03-设计模式

## 1. 适用边界
仅针对 URL 跳转 MVP，避免引入大而全框架。

来源：PRD《1.产品目标》《3.1 本期必做》

## 2. 推荐模式

### 2.1 命令模式（Command）
用于承接 `Go To Endpoint By URL` 入口。

来源：PRD《3.1 F1》《4.交互说明》

### 2.2 策略模式（Strategy）
用于路径解析策略分离：
1. Spring 类级注解解析策略
2. Spring 方法级注解解析策略

来源：PRD《3.1 F2 最小框架支持》

### 2.3 管道模式（Pipeline）
将流程拆分为：输入 -> 扫描索引 -> 匹配 -> 选择 -> 跳转。

来源：PRD《4.交互说明》

## 3. 明确不建议
1. 不引入复杂 DI 容器。
2. 不引入事件总线。

理由：MVP 仅一条核心流程，复杂框架会增加维护成本。
