# 01-业务流程总览

## 1. 主流程：URL 搜索并跳转
1. 用户触发命令 `REST Toolkit: Go To Endpoint By URL`。
2. 用户输入 `/user/add` 或 `POST /user/add`。
3. 系统执行 endpoint 索引扫描（或使用缓存索引）。
4. 系统进行 URL 匹配。
5. 根据匹配结果执行：
- 唯一命中：直接跳转到方法定义。
- 多个命中：弹出候选列表，由用户选择后跳转。
- 未命中：提示“未匹配到接口方法”。

来源：PRD《3.1 F1 URL 到方法跳转》《4.交互说明》

## 2. 关键子流程

### 2.1 路径拼接子流程
1. 读取类级路径（例如 `/user`）。
2. 读取方法级路径（例如 `/add`）。
3. 组合为完整路径（`/user/add`）。

来源：PRD《3.1 F1》（类级路径 + 方法级路径）

### 2.2 匹配判定子流程
1. 比对输入路径与完整路径。
2. 若输入包含 METHOD，则优先按 METHOD + 路径联合匹配。
3. 产出 `唯一命中/多命中/未命中`。

来源：PRD《3.1 F1 输入支持与命中逻辑》
