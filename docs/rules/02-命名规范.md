# 02-命名规范

## 1. 命名目标
命名应直接对应 PRD 业务术语，降低沟通成本。

来源：PRD《1.产品目标》《3.1 F1 URL 到方法跳转》

## 2. 核心术语命名
1. `Endpoint`：表示可跳转的接口方法。
2. `ClassLevelPath`：类级路径。
3. `MethodLevelPath`：方法级路径。
4. `ComposedPath`：拼接后的完整路径。
5. `MatchResult`：匹配结果（唯一命中/多命中/未命中）。

来源：PRD《3.1 F1》《5.验收标准》

## 3. 文件命名
1. 命令处理：`*-command.ts`
2. 解析器：`*-parser.ts`
3. 匹配器：`*-matcher.ts`
4. 导航器：`*-navigator.ts`

## 4. 命令 ID 命名
1. 使用 `restToolkit.<verb><Target>` 风格。
2. MVP 核心命令：`restToolkit.gotoByUrl`。

来源：PRD《3.1 F1》

## 5. 禁止事项
1. 禁止使用与 PRD 无关的缩写（如 `rtk`, `ctl`）。
2. 禁止以 UI 细节命名业务核心对象（如 `InputItem` 代替 `Endpoint`）。
