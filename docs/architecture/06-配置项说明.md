# 06-配置项说明

## 1. 文档目标
定义 `0.0.1` 版本中已实现并可生效的配置项，避免文档与代码不一致。

来源：`package.json` `contributes.configuration.properties`

## 2. 已实现配置项总览

| 配置键 | 类型 | 默认值 | 作用范围 | 说明 |
|---|---|---|---|---|
| `restToolkit.ignoreGlobs` | `string[]` | `['**/target/**','**/build/**','**/.git/**','**/node_modules/**']` | 工作区级 | 索引扫描时排除的文件路径规则 |
| `restToolkit.contextPath` | `string` | `''` | 工作区级 | 匹配前剥离的上下文路径前缀，例如 `/api` |

## 3. 配置项详细说明

### 3.1 `restToolkit.ignoreGlobs`
1. 该配置影响索引扫描范围。
2. 推荐保留默认值，并按项目情况追加生成目录排除规则。
3. 修改后建议执行 `Spring API Toolkit: Refresh Index`。

### 3.2 `restToolkit.contextPath`
1. 当网关或服务带有统一前缀时可设置，例如 `/api`。
2. 输入 URL 与 endpoint 比对前会先去除该前缀。
3. 示例：
- 配置：`/api`
- 输入：`/api/group-robot/add`
- 可匹配：`/group-robot/add`

## 4. 生效机制
1. 工作区配置优先于用户全局配置。
2. 配置变更后，建议手动刷新索引以确保结果一致。

## 5. 非本版本配置（已移除）
以下配置项在当前代码中未实现，不应在本版本文档中宣称支持：
1. `restToolkit.scanRoots`
2. `restToolkit.excludeGlobs`
3. `restToolkit.primaryHotkeyEnabled`
4. `restToolkit.fallbackHotkeyEnabled`
5. `restToolkit.maxCandidates`
6. `restToolkit.logLevel`
